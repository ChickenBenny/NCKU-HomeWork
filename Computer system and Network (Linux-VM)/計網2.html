labv3.service 內容











[Unit]
Description= A description for this service uint.

[Service]
ExecStart= /usr/local/bin/lab3.sh
Restart=always

[Install]
WantedBy=multi-user.target

Description用來設定這個Systemd Service Unit設定檔案的描述，可以讓管理員在日後查看時還能想起這個Systemd Service Unit的目的是什麼。

ExecStart項目可以用來設定當這個Systemd Service Unit項目啟動時要執行的應用程式(需使用完整絕對路徑)

Restart項目可以設定ExecStart所執行的程式要在什麼樣的情況被關閉時，以相同的指令再重新執行一次

[Install]區塊中，只有WantedBy這一個設定項目，它可以用來指定一個Systemd Target Unit，讓目前正在設定的這個Unit，會在運行該Systemd Target Unit時，也跟著被運行。

GTP(GUID partition table)

實體硬體的分割區表的結構

  • 在MBR硬碟中，分割區資訊直接儲存於主開機紀錄（MBR）中
  • GPT分配64bits給邏輯塊位址，因而使得最大分割區大小在264-1個磁區成為可能。對於每個磁區大小為512位元組的磁碟，那意味著可以有9.4ZB（9.4×1021位元組）或8 ZiB個512位元組
  • 64位元Windows作業系統使用16,384位元組（或32磁區）作為GPT分割區表

File: addallusers.sh
























#!/bin/bash
filename=$1

while read line
do
        user_name=$(grep "\<$line\>" $filename | awk '{print $1}')
        password=$(grep "\<$line\>" $filename | awk '{print $2}')
        group_name=$(grep "\<$line\>" $filename | awk '{print $3}')
        sudo useradd ${user_name} #加入到使用者
        if [ ! -z "$group_name" ]; then # 第三行不為空
                array=(${group_name//,/ })
                for var in ${array[@]}
                do
                        sudo usermod -a -G $var $user_name # 加入到group中
                done

        fi
         echo "${user_name}:${password}" | sudo chpasswd

#       echo ${user_name}
#       echo ${password}
#       echo ${group_name}
done < ${filename}
